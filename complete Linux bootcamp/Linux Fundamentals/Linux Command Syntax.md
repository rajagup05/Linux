
## Linux Command Syntax

The standard syntax for a Linux command follows a consistent three-part structure: `command [options] [arguments]`. Each component must be separated by at least one blank space, and the entire structure is case-sensitive.

- **command**: The actual program or built-in instruction you want to execute (e.g., ls, cd, grep).
- **options**: Also called flags, these modify how the command behaves. They are usually optional and start with one or two dashes.
- **arguments**: The target or input that the command acts upon, such as file paths, text strings, or user names.

### Rules for options

- **Short-form options**: A single dash followed by a single letter (e.g., `-l`).
- **Stacked options**: You can combine multiple short options behind one single dash (e.g., instead of writing `ls -l -a`, you can write `ls -la`).
- **Long-form options**: Two dashes followed by a full word (e.g., `--all` or `--reverse`).
