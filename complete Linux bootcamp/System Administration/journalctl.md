
## journalctl

journalctl is the command-line utility used to query and view system logs generated by the systemd-journald service on modern Linux distributions. It provides a centralized, structured, and high-performance way to access kernel, service, and application logs.

- View all logs: Run `journalctl` by itself to see every log entry starting from the oldest. Press `q` to exit.
- View newest logs first: Use `journalctl -r` to reverse the order so the most recent events appear at the top.
- Follow logs in real-time: Use `journalctl -f` to continuously display new log entries as they happen (similar to `tail -f`).
- Extra verbose output: Run `journalctl -xe` to jump to the end of the logs and show extra troubleshooting info for recently failed services.

- Today's logs: `journalctl --since today`.
- Relative time: `journalctl --since "1 hour ago"`.

- View a specific service: Use `-u` followed by the unit name (e.g., `journalctl -u nginx.service`).

- Check disk usage: `journalctl --disk-usage`.
